# propor

### Overview

propor is an R package for creating data for Wurman-style maps, a form of cartogram in which polygons (e.g. circles, hexagons, squares) arranged in a grid are proportionally resized according to a specified variable and visualised along with the original polygons for reference.

### Installation

The propor package can be installed from Github using devtools:

    library(devtools)
    install_github("lbuk/propor")

### Use
Here is an example of propor in use:

    library(propor)

    # Load the sf dataset of housing stats at borough-level
    data(housing_london)

    # Generate the data for the Wurman-style map
    df <- propor(housing_london, w = "new_dwellings", max_var = "lp2021_target")

The data generated by propor can now be visualised:

    library(ggplot2)

    # Visualise the data as a map using ggplot2
    ggplot(df) + 
        geom_sf(aes(fill = polygon), lwd = 0.2, col = "#000000") + 
        geom_sf_text(aes(label = borough_lab), size = 3, fontface = "bold") +
        scale_fill_manual(values = c('cartogram' = "#59f9ff", 'original' = "#eeefef"), breaks = c("cartogram"), labels = c("")) +
        labs(title = "Did average annual net housing completions between 2020/21 and 2022/23 
    make the 2021 London Plan target?", caption = "Note: the blue hexagons are proportional to the percentage of the 2021 London Plan target achieved
    Data Source: propor; geogrid; Trust for London: https://trustforlondon.org.uk/data/new-housing-completions/", fill = "", x = "", y = "") +
        theme(legend.position = 'none',
              axis.line = element_blank(),
              axis.ticks = element_blank(),
              axis.text = element_blank(),
              plot.title = element_text(size = 10.75, face = "bold"),
              plot.caption = element_text(size = 9, face = "plain", hjust = 1),
              panel.background = element_blank(),
              panel.grid = element_blank())
